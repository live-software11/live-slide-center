## Vincoli Assoluti (non negoziabili)

1. **Tenant isolation:** ogni tabella con dati business ha `tenant_id` + RLS (`tenant_isolation` + `super_admin_all`). Violazione = bug critico.
2. **`apps/player/` NON deve esistere** — Room Player = route `/sala/:token` in `apps/web/` (PWA).
3. **`presentation_versions` e append-only** — mai UPDATE, ogni modifica = nuova riga.
4. **Dark mode only** — ambiente operativo regia.
5. **Zero stringhe UI hardcoded** — tutto via `t('chiave')` con coppia IT/EN nello stesso commit.
6. **Supabase Storage** per MVP — Cloudflare R2 solo quando egress > $50/mese.
7. **Mai mDNS da browser** — Agent registra IP al cloud, PWA lo interroga dal cloud.
8. **Mai contenuto file clienti visibile a super_admin** — solo metadati (GDPR).

## Database e RLS

### Helper functions (gia in DB)

```sql
public.app_tenant_id()   -- estrae tenant_id dal JWT (app_metadata / user_metadata)
public.is_super_admin()  -- true se role = 'super_admin' nel JWT
```

### Pattern RLS per ogni tabella

```sql
ALTER TABLE nome_tabella ENABLE ROW LEVEL SECURITY;
CREATE POLICY tenant_isolation ON nome_tabella FOR ALL USING (tenant_id = public.app_tenant_id());
CREATE POLICY super_admin_all ON nome_tabella FOR ALL USING (public.is_super_admin());
```

### Enforcement DB (trigger + RLS)

- `check_storage_quota()` — blocca INSERT su `presentation_versions` se quota storage superata
- `update_storage_used()` — aggiorna `tenants.storage_used_bytes` automaticamente
- `check_events_quota()` — blocca INSERT su `events` se max eventi/mese superato
- `check_rooms_quota()` — blocca INSERT su `rooms` se max sale/evento superato
- `current_tenant_suspended()` — (Fase 14) SECURITY DEFINER: legge `tenants.suspended` per il tenant corrente; usata in tutte le policy `tenant_isolation` per bloccare dati operativi a tenant sospesi (SELECT su `users`/`tenants` preservato)

### Query client

- RLS filtra automaticamente per tenant — **NON aggiungere `.eq('tenant_id', ...)` nel client**
- Sempre `limit()` sulle liste
- Join con sintassi embedded: `select('*, rooms(*)')`
- Delete con conferma UI a due passaggi e hint CASCADE (sala → sessioni+relatori)

Idempotente. Permette al PC sala di dichiarare la modalita (`live` = polling 60s + throttle download per non disturbare proiezione, `turbo` = polling 5s + concurrency 3 per setup, `auto` = comportamento standard 12s) e all'admin di vederla in tempo reale dalla `EventDetailView`.
14. `20260418010000_room_realtime_broadcast.sql` — Sprint B (GUIDA_OPERATIVA_v3 §2.B) — Trigger `broadcast_presentation_version_change_trg` su `presentation_versions` (INSERT/UPDATE/DELETE) e `broadcast_room_state_change_trg` su `room_state` (UPDATE). Le funzioni `SECURITY DEFINER` chiamano `realtime.send(payload, event, 'room:<room_id>', false)` per pubblicare i cambi su un canale Realtime Broadcast. Necessario perche' il Room Player NON ha sessione utente Supabase (auth via `device_token`), quindi le RLS `tenant_isolation` impedirebbero a `postgres_changes` di ricevere eventi. Topic `room:<uuid>` non enumerable (UUID v4 con 122 bit di entropia). `EXCEPTION WHEN OTHERS NULL` per non bloccare scritture applicative se Realtime e' giu'.
15. `20260418020000_move_presentation_to_session.sql` — Sprint G (GUIDA_OPERATIVA_v3 §3.B / §3.C) — RPC `rpc_move_presentation_to_session(p_presentation_id, p_target_session_id)` per spostare presentation tra sessioni dello stesso evento. Distinta da `rpc_move_presentation` (tra speaker) perche' (a) UX "scegli sessione" non richiede di pensare allo speaker, (b) supporta presentation senza speaker (admin upload), (c) `same_session_no_op` ritorna `{skipped:true}` invece di errore (bulk con N file mixed funziona). `speaker_id` resettato a NULL (lo speaker e' legato alla vecchia sessione). Audit `move_presentation_to_session`. Riusata anche da Sprint H (drag&drop tra sessioni).
16. `20260418030000_room_state_now_playing.sql` — Sprint I (GUIDA_OPERATIVA_v3 §3.E) — colonne `room_state.current_presentation_id uuid REFERENCES presentations(id) ON DELETE SET NULL` + `room_state.last_play_started_at timestamptz` + RPC SECURITY DEFINER `rpc_room_player_set_current(p_token, p_presentation_id)`. La RPC autentica il PC sala (anonimo) via hash `extensions.digest(p_token,'sha256')` confrontato con `paired_devices.pair_token_hash`, valida che la presentation appartenga a una sessione DELLA STESSA sala del device (no cross-room tampering), aggiorna atomico + log `activity_log(actor='agent', action='room_now_playing')`. `GRANT EXECUTE ... TO service_role` (chiamata SOLO dall'Edge `room-player-set-current`, mai dal client). Trigger broadcast `broadcast_room_state_change_trg` (Sprint B) propaga la modifica su `room:<roomId>` → admin vede l'update in <1s.
